## Open Source Philosophy: The Commons

We believe **software is cultural heritage**, not property to be exploited or hoarded.

### Why Open Source Matters

**Free and Open Source Software (FOSS) is:**

- **The digital commons** — Like public libraries or parks, available to all
- **A form of cooperation** — Thousands of people building on each other's work
- **Essential infrastructure** — Linux powers cloud computing; Debian powers countless organizations
- **Verifiable** — Code can be reviewed, security vetted, modified
- **Durable** — Not at mercy of a corporation's business decisions

**Proprietary software:**

- Is locked away (you can't inspect, modify, or understand it)
- Is fragile (dependent on company staying in business)
- Extracts value (licensing fees, lock-in, data collection)
- Creates information asymmetry (vendor knows your system; you don't)

### The Problem We Solve

The software industry generates enormous value, but most of it is **extracted as private profit** rather than **contributed to the commons**.

**Examples of extraction:**

- Proprietary operating systems that lock users in
- SaaS platforms that create artificial scarcity
- Surveillance systems that monetize user behavior
- Legacy systems that prevent organizations from competing

**We do it differently:**

- **50% of our annual profits** go to open-source projects
- **Employees dedicate 5–10% of time** to OS contributions
- **Knowledge is published freely** (field guides, research, case studies)
- **Tools we build are released** (when possible, under open licenses)

### Key FOSS Projects We Support

We focus on projects that:

1. **Serve essential public goods** (Linux, Debian, accessibility tools)
2. **Benefit underserved communities** (developing countries, non-profits)
3. **Align with regenerative values** (energy efficiency, sustainability)
4. **Demonstrate cooperative spirit** (welcoming contributors, transparent governance)

**Examples of projects we might fund:**

- **Linux** — The operating system powering global infrastructure
- **Debian** — Stable, democratic GNU/Linux distribution
- **GNOME Accessibility** — Making software usable for everyone
- **OpenStreetMap** — Community-built mapping (alternative to proprietary maps)
- **Energy monitoring tools** — Software for tracking and reducing digital waste
- **Django & Python** — Web frameworks enabling digital services

### How Open Source Advances Our Mission

**Open source is regenerative because:**

1. **Collaboration** — Built on spontaneous cooperation, not contracts
2. **Knowledge sharing** — Reduces duplication, accelerates innovation
3. **Equity** — Accessible to everyone, not just those who can afford licenses
4. **Longevity** — Code persists beyond any single company's life
5. **Transparency** — Code can be audited for privacy, ethics, efficiency

### Challenges We Address

**Challenge:** "Open source means giving away what we built."

**Response:** We built consulting expertise, not code. We release code to build reputation, access innovation, and contribute to the commons.

**Challenge:** "Companies exploit open source without contributing back."

**Response:** True. We deliberately allocate profits and commit employee time specifically to support OS projects.

**Challenge:** "Open source projects are hard to fund sustainably."

**Response:** We provide multi-year funding commitments, not one-off donations. This enables real development, not just maintenance.

### The Lean C.R.O.W.D. Approach

**Our open-source commitment:**

1. **Structured giving** — 50% of profit goes to vetted projects
2. **Sustained support** — Multi-year funding so projects can plan
3. **Employee contribution** — Staff dedicate significant time to OS work
4. **Knowledge publication** — We share research and field guides freely
5. **Infrastructure support** — We pay for hosting, bandwidth, and tools

**The Board approves** which projects receive funding, ensuring:

- Alignment with regenerative values
- Geographic and cause diversity
- Measurable impact
- Healthy governance in funded projects

### Open Source as a Career Path

For our staff:

- **Contributing to open source** is part of the job (not side project)
- **Becoming a maintainer** is career development
- **Speaking at OS conferences** is encouraged
- **Publishing patents/innovations** is valued

**This creates:**

- Development of advanced skills
- Visibility in global community
- Meaningful work beyond client projects
- Career portability
