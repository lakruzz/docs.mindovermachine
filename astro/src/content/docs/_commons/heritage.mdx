## Cultural Heritage: Software as Public Legacy

Software is increasingly humanity's **cultural heritage** — integral to how we work, learn, create, and connect.

Just as physical cultural heritage (libraries, monuments, art) is preserved for future generations, **digital cultural heritage deserves the same care**.

### Why This Matters

**The Problem:**

- Proprietary software **disappears** when companies fail
- Loss of source code means **decades of work become inaccessible**
- Digital preservation requires **long-term maintenance**
- Market incentives **don't support preservation** of older systems

**In contrast:**

- Open-source code **survives beyond any company's lifespan**
- Communities can **maintain and evolve** software indefinitely
- Knowledge is **preserved and transferable**
- Future generations can **learn from and build on** our work

### Examples of Cultural Heritage at Risk

**Proprietary Systems:**

- Adobe Flash content (vast amounts now inaccessible)
- Legacy corporate systems (expensive to maintain after company's end)
- Deprecated mobile apps (platform changes make them incompatible)
- Data in proprietary formats (locked away, unreadable without vendor tools)

**Open-Source Foundation:**

- Linux (40+ years of continuous development)
- Apache web server (powers >40% of websites)
- Repositories of code that survive their original creators

### Open Source as Preservation

**Why open source preserves cultural heritage:**

1. **Resilience** — Code survives company bankruptcy
2. **Transparency** — Future developers understand design decisions
3. **Evolvability** — Communities can adapt software to new contexts
4. **Access** — Anyone can run, study, modify the software
5. **Collaboration** — Global community shares maintenance burden

### Projects Worth Preserving

**We prioritize funding projects that:**

- **Serve public goods** (everyone benefits, not just paying customers)
- **Demonstrate longevity** (likely to be useful for decades)
- **Have active communities** (not abandoned or single-maintainer)
- **Exemplify openness** (transparent governance, welcoming contributors)

### The Heritage Principle

**We ask:** "Is this software part of humanity's cultural heritage?"

**Yes, if:**

- It will be useful for 10+ years
- Broader society benefits
- It's not just commercial convenience
- Future developers can learn from it
- It exemplifies cooperation or regenerative principles

### How We Measure Heritage Impact

**Beyond typical metrics:**

| Metric                  | Typical Use | Heritage Use                          |
| ----------------------- | ----------- | ------------------------------------- |
| **Downloads**           | Popularity  | Indicates reach across communities    |
| **Contributors**        | Activity    | Shows community health and diversity  |
| **Longevity**           | Age         | Projects 10+ years old are heritage   |
| **Forks & Derivatives** | Adoption    | Shows adaptability to new contexts    |
| **Accessibility**       | Inclusivity | Can diverse people use and modify it? |
